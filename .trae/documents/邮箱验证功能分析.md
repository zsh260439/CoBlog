# 计划：邮箱验证功能分析

## 问题分析

用户询问"为什么我的项目有邮箱验证？在哪"，我需要确定：
1. 项目中是否存在邮箱验证功能
2. 如果存在，在哪里
3. 如果不存在，需要实现哪些部分

## 搜索结果

### 存在的邮箱相关代码

#### 1. 前端表单 - `client/src/views/MessageView.vue` (第25-33行)
```typescript
const form = reactive({
  author: '',
  content: '',
  email: '',          // 邮箱字段
  qq: '',
  isPrivate: false,
  enableEmailNotice: true,  // 邮件提醒复选框
  useMarkdown: true,
})
```

#### 2. Composable - `client/src/composables/useGuestbook.ts`
- `MessageFormData` 接口包含 `email` 和 `enableEmailNotice` 字段
- `submitMessage` 函数将 email 发送到后端

#### 3. API 调用 - `client/src/servers/message.ts`
- `CreateMessagePayload` 接口包含 `email?: string`

### 不存在的内容

1. **后端服务器代码** - `server/src/` 目录不存在（没有实际的后端代码）
2. **邮箱验证逻辑** - 没有找到任何邮箱格式验证
3. **邮件发送库** - 没有 nodemailer、sendgrid 等邮件库
4. **SPEC.md 说明** - 文档中没有提到邮箱验证功能

## 关键发现

### 邮箱输入框代码 - `client/src/views/MessageView.vue` (第82-87行)
```vue
<input
  v-model="form.email"
  class="composer-form__input"
  type="text"
  placeholder="邮箱"
/>
```
**这只是普通的文本输入框，没有任何前端验证！**

### 前端提交逻辑 - `client/src/composables/useGuestbook.ts`
```typescript
const handleSubmit = async () => {
  if (!form.author.trim()) { return }  // 只验证昵称
  if (!form.content.trim()) { return }  // 只验证内容
  const success = await submitMessage(form)
  // 邮箱没有任何验证！
}
```

### 后端请求 - `client/src/http-utils/index.ts`
```typescript
const httpInstance = axios.create({
  baseURL: 'http://localhost:3000/',  // 发送到后端
  timeout: 10000
})
```

## 结论

**项目中没有实现邮箱格式验证！** 邮箱输入框：
- `type="text"` (不是 `type="email"`)
- 没有任何正则表达式验证
- 提交时也不验证邮箱格式

## 可能的原因

1. **后端 API 返回的错误** - 你发送到 `http://localhost:3000/messages` 的请求
2. **如果后端不存在** - 会是 "Connection refused" 错误
3. **检查你的浏览器开发者工具** (F12) → Network 面板，看看具体返回什么错误
