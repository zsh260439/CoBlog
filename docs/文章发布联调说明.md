## 当前文章流转

### 发布链路

后台页：

- `/admin/article/new`

前端提交字段：

- `title`
- `slug`
- `content`
- `excerpt`
- `summary`
- `category`
- `categorySlug`
- `tags`
- `coverImage`

后端接收接口：

- `POST /articles`

### 展示链路

真实文章数据现在会流到这些页面：

- 博客列表：`/blog`
- 文章详情：`/article/:slug`
- 分类页：`/category/:slug`
- 标签页：`/tag/:tag`
- 归档页：`/archive`

## 为什么要做 normalize

因为现在数据库里的文章数据可能有两种来源：

1. 你后面从后台新发的文章
2. 之前就存在、字段可能不完整的老文章

如果不做统一处理，前台容易出现这些问题：

- `tags` 为空时报错
- `summary` 缺失导致详情页空白
- `wordCount` 缺失导致统计显示异常
- `coverImage` 为空字符串导致样式或校验问题

现在前后端两边都做了兜底，所以容错会高很多。

## 后台增强补充

这轮又补了后台内容管理的几个关键能力：

- 后端新增了 `taxonomy` 模块

对应文件：

- `client/src/views/admin/AdminCategoriesView.vue`

对应文件：

### 4. 路径自动生成

- 标题会自动生成基于拼音的 slug

对应文件：

- `client/src/utils/slug.ts`
